# Full Stack Engineer Rubric

## Evaluation Criteria for Full Stack Engineer Roles

### Correctness (Weight: 30%)
- Frontend and backend integration is correct
- API contracts are properly implemented
- Data flows correctly through the stack
- State management is appropriate

### Complexity (Weight: 15%)
- Frontend rendering performance is considered
- API response times are optimized
- Database queries are efficient
- No N+1 query problems

### Edge Cases (Weight: 15%)
- Loading states and error handling on frontend
- API error responses are properly handled
- Form validation on both client and server
- Authentication and authorization edge cases
- File upload handling
- Concurrent user scenarios

### Readability (Weight: 10%)
- Consistent naming across frontend and backend
- Component structure is logical
- API endpoints are RESTful or follow conventions
- CSS/styling is organized

### Maintainability (Weight: 10%)
- Clear separation between frontend and backend
- Shared types/interfaces are defined
- Environment configuration is externalized
- Code is modular and reusable

### Communication (Weight: 20%)
- API documentation is present
- Component documentation
- Code comments for complex business logic
- README or setup instructions

## Seniority Expectations

### Junior
- Can build basic CRUD applications
- Understands frontend-backend communication
- Learning state management

### Mid
- Builds complete features end-to-end
- Handles authentication and authorization
- Optimizes performance bottlenecks
- Implements proper error handling

### Senior
- Architects full-stack applications
- Makes technology stack decisions
- Handles scalability across the stack
- Implements CI/CD and monitoring
- Considers security across all layers
