# Algorithm Problem Rubric

## Evaluation Criteria for Algorithm Problems

### Correctness (Weight: 30%)
- Solution produces correct output for all test cases
- Algorithm handles the general case properly
- Logic is sound and complete
- No off-by-one errors or indexing mistakes

### Complexity (Weight: 15%)
- Time complexity is optimal or near-optimal for the problem
- Space complexity is justified
- No unnecessary data structures
- Algorithm choice matches problem constraints

### Edge Cases (Weight: 15%)
- Empty input (empty array, empty string, null)
- Single element input
- Maximum size input
- Duplicate values
- Negative numbers
- Sorted and unsorted inputs
- Boundary values (0, -1, INT_MAX)

### Readability (Weight: 10%)
- Variable names are descriptive
- Algorithm logic is clear and follows a pattern
- Helper functions are used where appropriate
- Code is not overly clever at the expense of clarity

### Maintainability (Weight: 10%)
- Solution is modular
- Functions are single-purpose
- Easy to modify for variations of the problem
- No hardcoded values that should be parameters

### Communication (Weight: 20%)
- Comments explain the algorithm approach
- Time/space complexity is stated
- Edge case handling is documented
- Variable names communicate intent

## Scoring Guide

### Excellent (90-100)
- Optimal solution with all edge cases handled
- Clean, well-documented code
- Correct complexity analysis
- Thoughtful variable naming

### Good (75-89)
- Correct solution with minor edge case gaps
- Readable code with some documentation
- Reasonable complexity

### Fair (60-74)
- Solution works for basic cases
- Missing several edge cases
- Code is functional but not well-organized

### Poor (below 60)
- Incorrect solution
- No edge case handling
- Poor code quality
- Syntax errors
