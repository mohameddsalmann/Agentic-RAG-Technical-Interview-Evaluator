# Backend Engineer Rubric

## Evaluation Criteria for Backend Engineer Roles

### Correctness (Weight: 30%)
- Solution correctly implements the required functionality
- API endpoints handle all specified HTTP methods correctly
- Data validation and error handling are present
- Business logic is accurate and complete

### Complexity (Weight: 15%)
- Time complexity is appropriate for the problem constraints
- Space complexity is optimized
- Algorithm choice is justified for the use case
- No unnecessary computations or redundant operations

### Edge Cases (Weight: 15%)
- Empty input handling
- Null/None value handling
- Boundary condition checks
- Concurrent access considerations
- Rate limiting and throttling
- Database transaction handling
- Timeout and retry logic

### Readability (Weight: 10%)
- Clear and descriptive variable/function names
- Consistent code style and formatting
- Appropriate use of design patterns
- Code is self-documenting where possible

### Maintainability (Weight: 10%)
- Modular design with separation of concerns
- Functions are single-purpose and reasonably sized
- Dependencies are clearly defined
- Code is testable (dependency injection, pure functions)

### Communication (Weight: 20%)
- Code comments explain non-obvious logic
- API documentation is present
- Error messages are user-friendly
- Code structure communicates intent clearly

## Seniority Expectations

### Junior
- Basic correctness is sufficient
- Simple solutions are acceptable
- Edge case awareness is developing
- Clean code is expected but not perfect

### Mid
- Solid understanding of trade-offs
- Handles most edge cases
- Applies appropriate design patterns
- Considers performance implications

### Senior
- Demonstrates deep system thinking
- Handles complex edge cases and concurrency
- Considers scalability and maintainability
- Makes informed architecture decisions
- Considers security implications
